# UI components: shadcn/ui

- **Default stack:** Build the interface with **[shadcn/ui](https://ui.shadcn.com/)** (Radix primitives + Tailwind). Treat shadcn as the source of truth for buttons, forms, dialogs, navigation menus, sheets, tables, and other shared UI.
- **Do not** add parallel general-purpose component libraries (e.g. another full kit) for the same role. Prefer composing **existing** primitives from `components/ui` before inventing new base widgets.
- **Adding primitives:** Use the shadcn CLI for new registry components (e.g. `npx shadcn@latest add button`) so imports, `cn()`, and Tailwind tokens stay consistent with `components.json`.
- **App-specific UI:** Feature-level components live next to features or under `components/` as needed, but they should **compose** `@/components/ui/*` (or the alias your `components.json` defines), not reimplement accessible patterns by hand unless shadcn does not cover the case.
- **Styling:** Follow the project’s Tailwind + shadcn theme tokens (CSS variables / `tailwind.config` as generated by shadcn init). Avoid one-off inline styles that fight the design system.
- **Icons:** Use **[Iconify](https://iconify.design/)** (`@iconify/react` + `@iconify/icons-*` data packages, e.g. `@iconify/icons-mdi`) for UI and brand icons. `components.json` uses `iconLibrary: "iconify"` for this preference.
<!-- END:ui-component-rules -->

# Cal.com booking

- **`NEXT_PUBLIC_CAL_LINK`:** Event path only (e.g. `username/30min`). When unset, “Agendar sesión gratis” falls back to `#contacto`.
- **`NEXT_PUBLIC_CAL_ORIGIN`:** Optional. Defaults to `https://cal.com` for embed init. Set when using self-hosted or a custom Cal domain (see `.env.example`).
